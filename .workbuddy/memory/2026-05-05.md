# 2026-05-05 工作日志

## AgentNexus - Orchestrator 路由匹配 Bug 修复

### 问题
测试文件 test.txt 中，规划阶段输出只有 `research: DeepSeek V3 vs GPT-4o code generation benchmark comparison 2026-05-05`，但 FSM 错误地进入了 `code` 节点并生成了不必要的 Python 代码。

### 根因
`orchestrator.py` 中所有路由判断使用子字符串匹配 `any("code" in s.lower() for s in plan)`，会错误匹配到 "code generation" 中的 "code" 子串。

### 修复内容
将所有 7 处子字符串匹配改为前缀匹配：

**文件**: `agentnexus/agents/multi_agent/orchestrator.py`

1. 第 264 行: `any(s.lower().startswith("research:") for s in plan)`
2. 第 265 行: `any(s.lower().startswith("code:") for s in plan)`
3. 第 279 行: `if step.lower().startswith("research:"):`
4. 第 330 行: `any(s.lower().startswith("code:") for s in plan)`
5. 第 340 行: `if step.lower().startswith("code:"):`
6. 第 556 行: `any(s.lower().startswith("code:") for s in plan)`
7. 第 666 行: `any(s.lower().startswith("research:") for s in plan)`
8. 第 736 行: `any(s.lower().startswith("code:") for s in plan)`

### 验证
所有修改已通过文件读取验证，前缀匹配逻辑正确。

### 效果
- "DeepSeek V3 vs GPT-4o code generation benchmark" 不再误触 code 节点
- 只有规划阶段明确包含 `code:` 前缀的步骤才会进入 code 节点
- FSM 路由判断更加精确

## WebSearch 功能对比分析 (前期讨论)

对比了 AgentNexus 的 web_search (基于 Tavily) 与 ChatGPT-5、Perplexity 的差异：

| 维度 | ChatGPT-5 / Perplexity | AgentNexus |
|------|------------------------|------------|
| 搜索后端 | Bing/Google + 自建索引 | Tavily 单源 |
| 查询理解 | Classifier + 子查询分解 | 无（直接透传） |
| 内容获取 | 完整页面爬取 + 分块 | Tavily 片段 |
| 重排序 | Cross-Encoder | 无（使用 Tavily score） |
| 引用 | 句子级 [1][2] | URL 级 SourceClaim |
| 深度适配 | `search_complexity` 分类器 | 关键词匹配 (`_pick_depth`) |

**P0 推荐修复**: 在 Tavily 调用中启用 `include_raw_content=True` 以获取完整内容而非仅片段。
